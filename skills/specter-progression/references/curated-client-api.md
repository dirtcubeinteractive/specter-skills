## Achievements

### Force Complete Task

`POST /v2/client/achievements/force-complete`

Forces the completion of a specified task, with an option to retrieve reward data in the response.

**Required headers:** `api-key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `taskIds` | array<string> | ✅ |  | Array containing a single task ID to force completion. Must contain exactly one ID. |
| `getRewardDataInResponse` | boolean | — |  | Flag to include reward data in the response if set to true. |


### Grant Multiple Custom Rewards

`POST /v2/client/achievements/grant-reward-batch`

Grants rewards directly without reference to a source. Supports batch operations for multiple reward types.

**Required headers:** `api-key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `rewards` | object | ✅ |  | Object containing rewards grouped by type. |
| `bypassLockCondition` | boolean | — |  | Flag to bypass lock condition. |
| `bypassLimitedEdition` | boolean | — |  | Flag to bypass limited edition restriction. |
| `customParams` | object | — |  | Custom parameters for processing. |
