## Real Money Gaming

### Create Order

`POST /v2/client/rmg/create-order`

Creates a new order for a real money gaming transaction using the provided amount, currency, and payment gateway name.

**Required headers:** `Api-Key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `amount` | number | ✅ |  | The order amount to be processed. Must be provided as an integer. |
| `currencyId` | string | ✅ |  | The identifier for the currency used in the order. Must be provided as a string. |
| `gatewayName` | string | ✅ |  | The name of the external payment gateway to be used for processing the order. |


### Execute Deposit

`POST /v2/client/rmg/execute-deposit`

Processes a deposit transaction by executing the deposit with the provided amount, currency, and external payment reference.

**Required headers:** `Api-Key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `amount` | number | ✅ |  | The deposit amount to be processed. Must be provided as an integer. |
| `currencyId` | string | ✅ |  | The identifier for the currency used in the deposit transaction. Must be provided as a string. |
| `referenceId` | string | ✅ |  | The unique reference identifier received from the external payment gateway when an order is placed. |


### Execute Withdrawal

`POST /v2/client/rmg/execute-withdrawal`

Processes a withdrawal transaction by executing the withdrawal with the provided amount, currency, optional wallet type, and external payment reference.

**Required headers:** `Api-Key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `amount` | number | ✅ |  | The withdrawal amount to be processed. Must be provided as an integer. |
| `currencyId` | string | ✅ |  | The identifier for the currency used in the withdrawal transaction. Must be provided as a string. |
| `walletType` | string | — | `deposit` | Optional wallet type for the withdrawal. Allowed value: 'deposit'. |
| `referenceId` | string | ✅ |  | The unique reference identifier received from the external payment gateway when a withdrawal order is placed. |


### Get Policies

`POST /v2/client/rmg/get-policy`

Fetches policies for a given RMG currency, covering all wallet types such as deposit, withdrawal, bonus wallets, as well as additional policies like taxation and priority.

**Required headers:** `Api-Key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `currencyId` | string | ✅ |  | The identifier of the RMG currency for which the policies will be fetched. |
| `entityType` | string | — |  | Optional type of entity to filter policies (e.g., deposit, withdrawal, bonus). |
| `entitySubType` | string | — |  | Optional subtype for further filtering policies if required. |

## Wallet

### Update Balance

`POST /v2/client/wallet/update-balance`

Updates the balance of a specified wallet by adding or subtracting an amount for a given currency.

**Required headers:** `api-key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `amount` | number | ✅ |  | The amount to add or subtract from the wallet balance. |
| `customParams` | object | — |  | Custom parameters for additional processing. |
| `currencyId` | string | ✅ |  | The id of the currency to update |
| `operation` | enum | ✅ | `add` \| `subtract` | The update operation to execute |
| `walletType` | enum | ✅ | `virtual` \| `rm_deposit` \| `rm_winning` \| `rm_bonus` | The type of the wallet to update (mainly relevant when updating RMG wallets. Default to virtual). |
