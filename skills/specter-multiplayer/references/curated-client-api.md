## Matches

### Create Match Session

`POST /v2/client/matches/create-session`

Creates a new match session, allowing participants to join and engage in the specified match.

**Required headers:** `api-key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `matchId` | string | ‚úÖ |  | The unique identifier of the match for which the session is being created. |

### Get Match Session Details

`POST /v2/client/matches/get-session-details`

Retrieves details for specific match sessions or, if no match session IDs are specified, returns match sessions filtered by competition IDs, game IDs, match IDs, or a combination of these filters. Supports optional pagination.

**Required headers:** `api-key`, `Authorization`

| Param | Type | Required | Allowed values | Description |
|---|---|---|---|---|
| `matchSessionIds` | array<string> | ‚Äî |  | Array of match session IDs to retrieve specific match session details. If provided, other filters are ignored. |
| `competitionIds` | array<string> | ‚Äî |  | Array of competition IDs to filter match sessions by competition. |
| `matchIds` | array<string> | ‚Äî |  | Array of match IDs to filter match sessions by specific matches. |
| `gameIds` | array<string> | ‚Äî |  | Array of game IDs to filter match sessions by specific games. |
| `offset` | number | ‚Äî |  | Pagination offset for the query to skip a specified number of results. |
| `limit` | number | ‚Äî |  | Pagination limit for the number of match session details to retrieve. |
